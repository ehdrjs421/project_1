# E‑Commerce Data Analysis
## [1] 프로젝트 개요 (Project Overview)
+ 분석 상황 (Context): 브라질 이커머스 플랫폼 'Olist'의 고객 만족도 조사 결과, 저평점(1~2점) 리뷰에서 배송 관련 불만이 반복적으로 관찰됨. 고객 경험(CX) 관점에서의 근본적인 원인 파악과 관리 체계 수립이 시급함.
+ 분석 목표 (Objectives): 
  + **고객 만족도 임계점 도출**: 배송 기간과 리뷰 평점 간의 상관관계를 분석하여, 만족도가 급격히 하락하는 핵심 임계 구간(Threshold)을 데이터로 특정함.
  + **프로세스 병목 지점 파악**: 장기 배송이 발생하는 핵심 원인을 단계별(출고, 배송 등)로 분해하여, 실제 개선이 필요한 물류 프로세스의 병목 지점을 특정함.
  + **기대 관리(Expectation Management)의 영향력 검증**: '절대적 배송 기간'과 '예정일 준수 여부'가 고객 심리에 미치는 상대적 임팩트를 비교하여, 정보 제공의 중요성을 증명함.
+ 핵심 오디언스: Olist CX팀 (고객 경험 기획 및 관리 부서)
## [2] 핵심 분석 결과 (Key Findings)
+ **인사이트 1:만족도 급락의 임계점, '배송 기간 20일'**
+ 데이터 분석 결과, 배송 기간이 늘어남에 따라 평점이 선형적으로 하락하는 것이 아니라,** 20일을 기점으로 만족도가 급격히 붕괴**되는 양상을 보였습니다.
+ **인사이트 2: 2장기 배송의 병목 구간, '출고 프로세스'의 지연**
+ 20일 이상 소요되는 장기 배송 건이 집중된 카테고리를 심층 분석한 결과, 물류 이동 시간보다 셀러의 출고 단계(Shipping Stage)에서 발생하는 지연이 최종 배송 기간과 상관관계를 보임
+ **인사이트 3:'장기 배송' 자체보다 '예측 불가능성'이 핵심 문제**
+ 배송이 20일 이상 걸리더라도, 배송 예정일(ESTIMATE)보다 빨리 온다면 리뷰 평점이 긍정적인 것을 확인

## [3] 프로젝트 구조 및 설정 (Setup & Architecture)

### 📂 폴더 구조
- `data/`: 로컬 원본 데이터 및 전처리된 데이터 저장 (Git 제외)
- `docs/meetings/`: 팀별 회의록 아카이빙 ([TEMPLATE.md](./docs/meetings/TEMPLATE.md) 활용)
- `notebooks/`: EDA 및 시각화 리포팅 (.ipynb)
- `src/`: 데이터 전처리 엔진 및 유틸리티 함수 (.py)
- `reports/figures/`: 최종 분석 결과 시각화 차트

### 🔧 환경 설정
본 프로젝트는 `Python 3.10+` 환경에서 수행됩니다.

```bash
# 가상환경 설정 및 패키지 설치
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## [4] 협업 규칙 (Collaboration & Git Workflow)

### 💬 커뮤니케이션
- **회의록**: 모든 기록은 `./docs/meetings/` 폴더에 `YYYY-MM-DD_제목.md` 형식으로 관리합니다.
- **코드 리뷰**: `main` 브랜치 보호를 위해 반드시 Pull Request 후 승인을 거쳐 Merge 합니다.

### 🚩 Git 커밋 컨벤션
- `[feat]`: 새로운 기능 추가 (전처리 로직 등)
- `[fix]`: 버그 수정
- `[docs]`: 문서 수정 (README, 회의록 등)
- `[refactor]`: 코드 구조 개선
